# 贡献指南

感谢你对 DRD 方法论项目的关注！我们欢迎任何形式的贡献。

## 如何贡献

### 报告问题

如果你发现了 Bug 或有功能建议，请：

1. 检查 [Issues](https://github.com/your-username/drd-methodology/issues) 中是否已有相关问题
2. 如果没有，创建新的 Issue，详细描述问题或建议

### 提交代码

如果你想提交代码：

1. Fork 本仓库
2. 创建你的特性分支 (`git checkout -b feature/AmazingFeature`)
3. 提交你的更改 (`git commit -m 'Add some AmazingFeature'`)
4. 推送到分支 (`git push origin feature/AmazingFeature`)
5. 创建一个 Pull Request

### 提交文档

DRD 是一个方法论项目，文档贡献尤为重要：

- 纠正文档中的错误
- 补充更多实践案例
- 添加检查清单模板
- 提供不同语言的翻译

## 代码风格

- 使用中文编写文档
- 保持格式清晰易读
- 使用 Markdown 语法规范

## 行为准则

- 尊重所有贡献者
- 保持专业和建设性的交流
- 避免敏感话题和争议性内容

## 许可证

通过贡献代码或文档，你同意你的贡献将在 MIT 许可证下发布。
