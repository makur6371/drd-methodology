# DRD 交互式体验程序

## 🎯 简介

这是一个交互式命令行程序，让你能够亲身体验**对话精炼开发法 (DRD)** 方法论的威力。

## 🚀 快速开始

```bash
# 进入项目目录
cd drd-experience

# 安装依赖（可选，目前只使用 Node.js 内置模块）
# npm install

# 运行程序
npm start
```

## 📖 体验模式

### 1. 完整体验 🎮

从头开始体验 DRD 方法论的 5 个阶段：
- 阶段 1：愿景澄清
- 阶段 2：基础调研
- 阶段 3：深度设计
- 阶段 4：需求精炼
- 阶段 5：方案固化

### 2. 示例演示 📖

查看 nflow 项目的完整案例演示，了解 DRD 方法论在实际项目中的应用。

### 3. 随机阶段 🎲

体验单个阶段，快速了解某个特定环节。

## 💡 使用提示

- 使用箭头键或直接输入数字选择选项
- 按回车键确认输入
- 输入 `n` 可以暂停或退出当前体验
- 随时返回主菜单

## 🎓 学习目标

通过体验程序，你将了解：

1. **对话驱动**的重要性
2. **外部调研**的价值
3. **迭代精炼**的威力
4. **需求精炼**的方法
5. **方案固化**的标准

## 📝 示例项目：nflow

将 iflow-cli 改造为 AI 小说写作工具的完整案例：

- 对话轮次：约 20 轮
- 产出文档：4 个完整文档
- 文档规模：约 15000 字
- 核心成果：
  - 6 个核心 Agent 设计
  - 字数控制系统（2000-2500字）
  - 质量把控闭环（审核→检查→修复→评分）
  - 自动修复机制（6种问题类型）
  - 零成本方案（iflow 免费模型）
