# DRD 体验程序使用说明

## 🚀 快速启动

```bash
# 进入项目目录
cd drd-experience

# 运行程序
node index.js

# 或者使用 npm
npm start
```

## 📖 程序功能

### 主菜单

程序启动后，会显示欢迎信息和主菜单：

```
============================================================
🔥 对话精炼开发法 (DRD) - 交互式体验程序
============================================================

📚 什么是 DRD？
对话精炼开发法是一种创新的 AI 辅助项目开发方法论
通过多轮深度对话 + 外部调研 + 迭代精炼
将模糊愿景转化为可执行方案

🎯 核心价值：
  🎯 需求精准落地 - 从模糊愿景到具体指标
  🔍 问题早期发现 - 通过对话暴露潜在问题
  📊 方案可执行 - 分阶段、可量化、可验证
  🚀 提高成功率 - 减少返工，降低风险

============================================================
请选择体验方式：
============================================================
1. 🎮 完整体验 - 从头开始体验 DRD 方法论
2. 📖 示例演示 - 查看 nflow 项目的完整案例
3. 🎲 随机阶段 - 体验单个阶段
4. ❌ 退出
```

### 1. 完整体验 🎮

**适用场景**：想要完整体验 DRD 方法论的 5 个阶段

**操作步骤**：
1. 选择 `1` 进入完整体验模式
2. 输入项目名称（或按回车使用示例项目 nflow）
3. 按照提示回答每个阶段的问题
4. 可以随时输入 `n` 暂停体验

**体验流程**：

```
阶段 1: 愿景澄清
────────────────────────────────────────────
📝 明确项目定位和核心价值

❓ 你想做什么？
你的回答: [你的回答]

❓ 为什么做这个项目？
你的回答: [你的回答]

...

────────────────────────────────────────────
📊 本阶段产出：
  1. 项目定位
  2. 核心价值
  3. 对标项目

继续下一阶段？(y/n): y
```

**5 个阶段**：

1. **愿景澄清** - 明确项目定位和核心价值
2. **基础调研** - 了解现有技术和竞品
3. **深度设计** - 设计核心解决方案
4. **需求精炼** - 将抽象需求转化为可执行指标
5. **方案固化** - 形成完整的可执行方案

### 2. 示例演示 📖

**适用场景**：想要了解 DRD 方法论在实际项目中的应用

**操作步骤**：
1. 选择 `2` 进入示例演示模式
2. 按回车键开始演示
3. 自动展示 nflow 项目的完整案例

**演示内容**：

- 项目描述：将 iflow-cli 改造为 AI 小说写作工具
- 核心价值：靠谱、免费、强大
- 对话轮次：约 20 轮
- 产出文档：4 个完整文档
- 文档规模：约 15000 字

**核心成果**：
- 6 个核心 Agent 设计
- 字数控制系统（2000-2500字）
- 质量把控闭环（审核→检查→修复→评分）
- 自动修复机制（6种问题类型）
- 零成本方案（iflow 免费模型）

### 3. 随机阶段 🎲

**适用场景**：想要快速了解某个特定阶段

**操作步骤**：
1. 选择 `3` 进入随机阶段体验模式
2. 选择阶段编号（1-5）
3. 选择是否使用示例项目演示

**可用阶段**：

```
1. 愿景澄清 - 明确项目定位和核心价值
2. 基础调研 - 了解现有技术和竞品
3. 深度设计 - 设计核心解决方案
4. 需求精炼 - 将抽象需求转化为可执行指标
5. 方案固化 - 形成完整的可执行方案
```

### 4. 退出 ❌

选择 `4` 或在任何提示时输入 `n` 即可退出程序。

## 💡 使用提示

### 交互技巧

1. **快速体验**：直接按回车键使用默认值或示例项目
2. **暂停体验**：在任何提示时输入 `n` 可以暂停或退出
3. **返回菜单**：体验完成后可以选择返回主菜单或退出
4. **多次体验**：可以反复体验不同的阶段和模式

### 学习建议

1. **先看示例**：建议先选择"示例演示"了解完整流程
2. **分步体验**：可以选择"随机阶段"逐个了解每个阶段
3. **完整实践**：最后选择"完整体验"从头开始实践
4. **记录心得**：体验过程中可以记录问题和想法

## 🎓 学习目标

通过体验程序，你将了解：

1. **对话驱动**的重要性
   - 如何通过对话澄清模糊需求
   - 如何提出有效的关键问题
   - 如何从回答中提取核心信息

2. **外部调研**的价值
   - 如何分析对标项目
   - 如何识别竞品和痛点
   - 如何确定技术路线

3. **迭代精炼**的威力
   - 如何从抽象概念到具体方案
   - 如何应对技术挑战
   - 如何控制项目成本

4. **需求精炼**的方法
   - 如何量化抽象需求
   - 如何建立质量控制流程
   - 如何设计自动修复机制

5. **方案固化**的标准
   - 如何构建文档体系
   - 如何统一命名规范
   - 如何制定开发指令

## 🔧 技术说明

### 运行环境

- Node.js 14+
- 无需额外依赖
- 支持 Windows、macOS、Linux

### 文件结构

```
drd-experience/
├── index.js         # 程序入口
├── interactive.js   # 交互逻辑
├── stages.js        # 阶段定义和示例数据
├── package.json     # 项目配置
├── README.md        # 项目说明
└── USAGE.md         # 本文件
```

### 扩展开发

如果你想添加新的示例项目或阶段，可以修改 `stages.js` 文件：

```javascript
export const STAGES = {
  // ... 现有阶段
};

export const EXAMPLE_PROJECT = {
  name: '你的项目名称',
  description: '项目描述',
  stages: {
    1: {
      answers: {
        '问题1': '答案1',
        // ...
      },
      outputs: {
        '产出1': '内容1',
        // ...
      }
    },
    // ... 其他阶段
  }
};
```
